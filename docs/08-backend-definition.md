# 08. バックエンド定義

## 目的

本書は、既存の要件・画面設計・状態遷移を前提に、v1 でバックエンドが担う責務を定義するための文書である。
ここでいうバックエンドとは、外部サーバーではなく、`VS Code Extension Host` 上で動作するアプリケーション層・文脈収集層・モデル呼び出し層・ナレッジ層を指す。

## 前提

- v1 では専用のリモートバックエンドは持たない
- AI 呼び出しは `GitHub Copilot + VS Code Language Model API` を利用する
- 保存先はローカル環境であり、個人用データのみを扱う
- UI は `WebviewView` を利用し、バックエンドは UI に対して状態と実行結果を提供する

## バックエンドの責務

| 領域 | 責務 | 対応する仕様 |
|---|---|---|
| 接続管理 | Copilot 利用可否判定、接続開始、同意フロー通過、接続状態更新 | 02.2, 04, 06 |
| 状態管理 | 画面表示用の状態を一元管理し、UI に返す | 05, 06, 07 |
| 文脈収集 | 送信候補となる作業文脈を収集し、除外条件を適用する | 02.1, 02.4, 03.1, 03.2 |
| 送信計画 | 今回送る文脈、送信理由、送信量、会話に紐付ける参照文脈を組み立てる | 02.6, 03.1, 03.3, 05.3 |
| AI 実行 | 手動相談・常時助言・会話継続の質問をモデルへ送信し、応答を返す | 02.1, 02.2, 02.3, 05.2, 05.3 |
| 発話制御 | 間隔、アイドル時間、重複抑制、一時停止を制御する | 02.7, 05.2, 05.3, 05.5, 06 |
| 会話管理 | 会話履歴、最新アドバイス、回答に紐付く参照文脈を保持する | 05.2, 05.3, 05.7 |
| ナレッジ管理 | 個人ナレッジの保存、検索、閲覧、削除、元会話リンク、回答生成時の再利用を行う | 02.5, 03.4, 03.6, 05.4 |
| 設定管理 | 初期モード、アイドル時間、除外 glob を保存・適用する | 03.3, 05.5 |
| 失敗時制御 | 制限状態への遷移、再試行可否判定、回復後の復帰処理を行う | 03 失敗時挙動, 06 |

## バックエンドの実装対象

### 8.1 接続管理

バックエンドは以下を提供する。

- 現在状態の取得
- `Copilotに接続` 実行
- `Trusted Workspace`、モデル取得可否、同意結果の判定
- 初回接続 probe の実行
- 接続失敗理由の分類

必要な内部処理は以下とする。

- `disconnected / connecting / consent_pending / connected / restricted / unavailable` の状態管理
- 接続開始中の多重実行防止
- 同意キャンセル時の未接続復帰
- quota 超過、一時制限、offline、untrusted の分類

### 8.2 画面状態の集約

バックエンドは UI ごとの生データではなく、画面表示可能な ViewModel を返す。

- S01 用: 接続状態、機能概要、利用条件案内、エラー文言
- S02 用: モード、入力可否、選択範囲などの簡易文脈、クールダウン状態
- S04 用: 会話履歴、入力可否、選択範囲などの簡易文脈、回答アクション、クールダウン状態
- S05 用: ナレッジ一覧、詳細、検索結果、元会話情報、削除可否
- S06 用: 現在設定値、有効値、保存可否
- S07 用: エラー種別、説明、推奨アクション、再試行可否
- S08 用: 相談履歴一覧、選択中の相談、削除可否

### 8.3 文脈収集

バックエンドは、AI 送信前に以下の文脈を収集する。

- アクティブファイル断片
- 選択範囲
- diagnostics
- 最近の編集範囲
- 関連シンボル候補
- 追加コンテキスト

文脈収集では以下を満たす。

- 保護済み除外 glob とユーザー指定 glob に従って文脈を除外する
- 生成物、依存物、キャッシュ、DB、アーカイブ、秘密情報を含みやすいパスを保護済み除外 glob で除外する
- 会話や状態表示に使える最小限の文脈要約を持たせる
- 各文脈は固定上限で切り詰め、送信量が過大になりにくい形にする

現行 v1 では、ファイル構造、定義と参照、ワークスペース横断探索、Git 履歴ベースの最近の変更内容、本文スキャンによる秘密情報検出、ファイルサイズ実測による動的な大容量判定は必須要件に含めない。

### 8.3.1 現在実装している変更前後収集
現在コードでは、`ContextCollector` が開いているドキュメント本文をメモリ上に保持し、変更イベント時に直前状態を参照できるようにしている。
このスナップショットは以下のタイミングで更新される。

- 拡張初期化時に、すでに開いている `TextDocument` を取り込む
- ドキュメントを開いた時に本文を取り込む
- ドキュメント変更後に最新本文へ更新する
- ドキュメントを閉じた時に破棄する

変更イベント時には、`rangeOffset` と `rangeLength` を用いて変更前本文から該当範囲を切り出し、変更後テキストと組み合わせて最近の編集要約を生成する。
現在の表現形式は、`変更前「...」 -> 変更後「...」`、`追加「...」`、`削除「...」` のいずれかである。

この方式により、専用バックエンドや Git 差分に依存せず、現在セッション中の直前変更に対する助言が可能になる。
一方で、比較対象は `HEAD` や保存済みファイルではなく、あくまで拡張が保持している直前スナップショットである。

### 8.4 送信計画と参照文脈生成

バックエンドは、モデルに送る前に `RequestPlan` を構築する。

`RequestPlan` には最低限以下を含める。

- リクエスト種別
- 利用モード
- 採用された文脈の一覧
- 除外された文脈の一覧
- 推定送信量
- 会話履歴やナレッジ保存に紐付ける参照文脈
- 重複抑制判定用の特徴量

`RequestPlan` は内部制御と会話履歴への紐付けを主目的とする。
専用の送信範囲確認画面やアドバイス詳細画面で `RequestPlan` を表示することは、現行 v1 の必須要件に含めない。

リクエスト種別は以下を想定する。

- 初回接続 probe
- 手動相談
- 常時助言
- 会話継続の質問
- ナレッジ候補化のための要約

### 8.5 AI 実行

バックエンドは、用途に応じてプロンプトと送信コンテキストを組み立て、モデル呼び出しを実行する。

- 手動相談: ユーザーの明示操作で実行
- 常時助言: トリガー条件を満たしたときのみ実行
- 会話継続の質問: 既存の会話履歴を引き継いで実行

AI 実行で必要な要件は以下とする。

- 学習支援特化のプロンプトを固定方針として適用する
- 参照した文脈を応答結果に紐付ける
- 応答をストリーミング可能な形で UI に渡せるようにする
- 応答失敗時に状態遷移とエラー情報を返す

### 8.6 発話制御

常時モードのために、バックエンドはスケジューラを持つ。

- 内部固定の最小リクエスト間隔
- ユーザーが設定できるアイドル時間
- 直近応答との類似度判定
- 制限中の自動停止
- 一時停止 / 再開

トリガー候補は以下とする。

- テキスト編集
- アクティブエディタ変更
- diagnostics 変化
- アイドル時間経過

ただし、トリガー発火時に毎回送信するのではなく、`発火判定 -> 送信計画生成 -> クールダウン判定 -> 実行` の順で判定する。

### 8.6.1 現在実装している常時助言フロー

現在コードでは、`AdviceScheduler` と `NavigatorController` が常時助言を分担している。

- `NavigatorController` が VS Code のイベントを購読し、編集・選択・アクティブエディタ・diagnostics の変化を受け取る
- 変化を `AdviceScheduler` に渡し、待機中トリガーとして保持する
- `AdviceScheduler` は `idleDelayMs` と `requestIntervalMs` を判定し、条件を満たした時のみ発火する
- 発火後、`RequestPlanner` が送信対象と除外対象を決める
- `AdviceService` が学習支援向けプロンプトを組み立て、Copilot へ送信する
- 応答は会話履歴と最新アドバイスへ反映される

また、重複抑制は「直前に送信した自動助言の文脈特徴量」と比較する簡易方式で実装している。
特徴量には、アクティブファイル断片、選択範囲、diagnostics、最近の編集、関連シンボルを含める。

制限系エラーやモデルエラーが返った場合は、自動助言を停止し、必要時モードへ戻す。

### 8.7 会話とアドバイス履歴

バックエンドは以下を管理する。

- 会話履歴
- 最新アドバイスカード
- 回答に紐付く参照文脈
- ナレッジ候補化に必要な周辺会話

履歴 1 件あたりに保持するデータ例は以下とする。

- 種別: 手動相談 / 文脈付き相談 / 常時助言 / 会話継続
- 発生日時
- ユーザー入力
- AI 応答本文
- 参照文脈要約
- ナレッジ候補化状態

### 8.8 ナレッジ管理

バックエンドは SQLite を用いて個人ナレッジを管理する。

必要な機能は以下とする。

- 回答からのナレッジ作成
- ナレッジ検索
- ナレッジ詳細表示
- ナレッジ削除
- ナレッジ元会話の特定
- 現在文脈に関連するナレッジ候補の検索
- 送信量を抑えるための取り込み件数制御

最低限必要な保存項目は以下とする。

- `id`
- `title`
- `summary`
- `body`
- `status`
- `sourceAdviceId`
- `createdAt`
- `updatedAt`

`status` は保存項目として持つが、現行 v1 の UI では無効化 / 再有効化は扱わない。
また、`tags`、JSON / Markdown エクスポート、全件リセットは現行 v1 の必須要件に含めない。

加えて、AI 送信時には以下の処理を行う。

- 現在文脈に関連するナレッジ候補の検索
- キーワード一致による簡易スコアリング
- 上位数件だけをプロンプトに含める

### 8.9 設定管理

バックエンドは設定の読み書きと有効化を行う。

管理対象は以下とする。

- 初期モード
- アイドル時間
- 除外 glob

リクエスト間隔は内部固定値として扱い、現行 v1 の設定画面では直接編集しない。

以下は現行 v1 の設定画面では扱わない。

- リクエスト間隔の直接編集
- 常時モード有効可否
- 重複抑制
- 送信対象カテゴリ ON / OFF
- 大容量閾値
- ナレッジ再利用 ON / OFF
- 送信前確認 ON / OFF

設定は以下の 2 層で扱う。

- 永続設定: `workspaceState` または `globalState`、必要に応じて `vscode.workspace.getConfiguration`
- 実行時設定: 現在反映済みのスケジューラ・収集条件・UI 表示用設定

### 8.10 失敗時制御

バックエンドは、失敗時のエラーを UI 向けの状態へ正規化する。

扱うべき代表状態は以下とする。

- 同意未取得
- Copilot 利用不可
- offline
- quota 超過
- 一時エラー
- untrusted workspace

必要な処理は以下とする。

- S07 表示用のメッセージ生成
- 再試行可否判定
- 常時モードの自動停止
- 制限復旧後の必要時モード復帰

## バックエンド内部モジュール案

現行 v1 では以下のモジュール分割とする。

- `ConnectionService`: Copilot 接続と利用状態管理
- `SessionStore`: 画面状態、会話状態、最新アドバイスの保持
- `ContextCollector`: 文脈収集
- `RequestPlanner`: 除外条件適用、送信計画、参照文脈情報の生成
- `AdviceScheduler`: 常時モードのトリガー制御
- `AdviceService`: プロンプト生成、モデル呼び出し、応答整形、ナレッジ候補化
- `ConversationStore`: SQLite による相談履歴管理
- `KnowledgeStore`: SQLite CRUD と関連ナレッジ検索
- `SettingsService`: 設定の永続化と反映

`ErrorMapper` は現行実装では独立モジュール化せず、今後のエラー系改善タスクとして扱う。

## 現行実装で意図的に簡略化している点

現行 v1 では以下を仕様として扱う。

- 最近の編集範囲は直前スナップショット比較で扱い、Git 履歴や保存済み版との比較は行わない
- 関連シンボルは現在行や選択範囲からの簡易抽出で扱い、定義 / 参照の本格解析は行わない
- 除外条件適用は `RequestPlanner` に含め、`ContextFilter` は独立モジュール化しない
- 関連ナレッジ検索は `KnowledgeStore` に含め、`KnowledgeRetriever` は独立モジュール化しない

エラー系は未着手の改善領域として残す。

- offline、quota 超過、同意未取得、untrusted workspace などの詳細分類
- S07 向けの状態別メッセージ、推奨アクション、再試行可否の精緻化
- `ErrorMapper` の独立モジュール化

## バックエンドの非責務

v1 のバックエンドは以下を行わない。

- 外部サーバーとの同期
- チーム共有ナレッジ
- Copilot 認証情報の保存
- AI によるコード変更やコマンド実行
- Web 版 VS Code 対応
