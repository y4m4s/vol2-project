# 03. 送信データ・保存データ・失敗時挙動

## 送信データ仕様

### 3.1 送信範囲

- アクティブファイル断片
- 選択範囲
- diagnostics
- 最近の編集範囲
- 関連シンボル

現在実装では、`最近の編集範囲` は `VS Code Extension Host` 内で保持している直前スナップショットを用いて組み立てる。  
編集イベント発生時に、変更前本文から該当範囲を切り出し、変更後入力内容と組み合わせて `変更前「...」 -> 変更後「...」` の要約として扱う。

### 3.2 送信除外

- `.env`
- 秘密情報
- 生成物
- 大容量ファイル
- ユーザー指定 glob

### 3.3 送信方針

- 送信対象は、選択範囲表示や設定画面の除外パターンなどを通じて把握しやすくする
- ユーザーが除外設定を編集できるようにする
- 専用の送信範囲確認画面および毎回の送信前確認設定は v1 の必須要件に含めない

常時モードでは、編集イベントごとに即送信するのではなく、以下の順で準リアルタイム送信する。

- 編集 / 選択 / ファイル切替 / diagnostics 変化を検知
- 直前スナップショットから変更前後要約を生成
- アイドル時間経過を待つ
- クールダウンと重複抑制を判定する
- 条件を満たした場合のみ AI に送信する

## データ保存仕様

### 3.4 保存対象

- 個人ナレッジ
- 接続状態の補助情報
- 拡張設定の一部

### 3.5 保存方式

- 個人ナレッジはローカルの `SQLite` に保存する
- 保存先はローカル環境とする
- GitHub Copilot の認証情報やアクセストークンは保存しない

### 3.6 ナレッジ操作

- 編集
- 削除
- 無効化
- エクスポート
- リセット

## 失敗時挙動

- 同意未取得時は、ユーザーに再接続または再試行を促す
- Copilot 未契約または利用不可時は、セットアップ案内を表示する
- quota 超過時は、常時モードを停止し、必要時モード中心の動作に切り替える
- offline 時は、AI リクエストを停止し、再接続可能時に再試行できるようにする
- untrusted workspace 時は、文脈収集を伴う AI 利用を制限または停止する
