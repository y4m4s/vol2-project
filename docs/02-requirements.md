# 02. 要件

## 機能要件

### 2.1 作業文脈の取得とフィードバック

ユーザーの記述しているコードや、現在の作業文脈を取得し、リアルタイムまたは準リアルタイムでフィードバックやアドバイスを返す。
常時モードでは、固定の最小リクエスト間隔、ユーザーが設定できるアイドル時間、重複抑制、一時停止によって発話頻度を制御する。

### 2.2 モード切り替え

アドバイスは、常時出すモードと、必要なときのみ出すモードの切り替えを可能にする。
必要時モードでは、ユーザーの明示的な操作によって AI の助言を表示する。
常時モードは、初回接続完了後にのみ有効化できる。

### 2.3 対話 UI

VS Code 上の横ウィンドウ等で、ユーザーが AI と対話できる UI を提供する。
ユーザーはその場で質問し、会話画面上で続けて補足依頼ができる。
UI 上では、接続状態、現在の選択範囲などの簡易的な参照文脈を確認できるようにする。
専用のアドバイス詳細画面、深掘り専用導線、送信範囲確認画面は v1 の必須要件に含めない。

### 2.4 プロジェクト文脈を踏まえた回答

AI は単なる一般論ではなく、プロジェクトの文脈を踏まえて回答できるようにする。
現行 v1 では以下を参照対象とする。

- アクティブファイル断片
- 選択範囲
- diagnostics
- 直近編集イベントから生成した最近の編集要約
- 現在行や選択範囲から抽出した関連シンボル候補
- ユーザーが入力した追加コンテキスト

ファイル構造、定義と参照、ワークスペース横断探索、Git 履歴ベースの最近の変更内容は、現行 v1 の必須要件に含めない。
参照対象は送信除外設定およびユーザー設定に従って制御されるものとする。

### 2.5 個人用ナレッジの蓄積と再利用

作業中にユーザーが詰まった点や、その解決の過程、次回の注意点などを、個人用のナレッジとして蓄積し、以後の回答や提案に再利用できるようにする。
ナレッジはローカルの `SQLite` に保存し、一覧検索、詳細確認、削除、元になった会話への遷移、回答生成時の再利用を可能にする。

### 2.6 参照文脈の簡易表示

AI が助言を行う際には、可能な範囲で参照した作業文脈を応答や会話履歴に紐付ける。
ただし、ファイル、関数、エラー、変更差分、関連シンボルを専用の根拠一覧として表示する画面は v1 の必須要件に含めない。
UI では、選択範囲や会話に紐付く文脈の要約を必要最小限に表示する。

### 2.7 発話頻度制御

AI の発話頻度は過剰にならないよう制御する。
連続して話しかけすぎないための固定インターバル制御、ユーザーが設定できるアイドル時間、同じ内容の繰り返しを避ける仕組み、一時停止 / 再開の仕組みを持たせる。

### 2.8 ナレッジの管理単位

ナレッジは個人利用を前提とし、ユーザーごとに管理する。
初期段階ではチーム共有や他者との同期は必須要件に含めない。

### 2.9 学習支援に特化した応答

AI の応答は、学習支援を主目的とした内容にする。
正解だけを提示するのではなく、考えるための観点、切り分け方、確認ポイント、次に見るべき箇所などを重視する。
原則として、答えの代行や自動実行につながる振る舞いは行わない。

## 非機能要件

- 学習体験を阻害しないこと
- AI の介入は有用である一方、作業の集中を妨げすぎないこと
- 応答はできるだけ短時間で返すこと
- 速度のみを優先せず、文脈を踏まえた有用性とのバランスを取ること
- ユーザーが AI に送信される情報の範囲を把握しやすいこと
- ナレッジデータはローカル保存とし、外部同期を前提としないこと
